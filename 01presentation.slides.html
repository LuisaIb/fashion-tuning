Why was the Fashion-MNIST dataset created?

Isabel Janez
  • Original MNIST dataset: handwritten digits
  • MNIST is often used as benchmark to validate algorithms
  • MNIST is too easy, overused and can not represent modern Computer Vision tasks
  • Fashion-MNIST intended to serve as a direct drop-in replacement, sharing same images size and the structure of training and testing splits

The aim of Fashion-MNIST dataset

  • Good benchmark dataset
  • Accessibility of MNIST: small size, straightforward encoding and permissive license
  • A more challenging classification task

  • Fashion-MNIST: a Novel Image Dataset for Benchmarking Machine Learning Algorithms. Han Xiao, Kashif Rasul, Roland Vollgraf. arXiv: cs.LG/1708.07747

Metrics Judith Romer

Accuracy

  • Measures the proportion of correctly classified data

  • $\text{Accuracy} = (\frac{\text{Number of correct predictions}}{\text{Total number of predictions}}) * 100$

  • Simple interpretation - especially for balanced class distribution

  • Balanced Accuracy not required as data classes are balanced

Cross Entropy Judith Romer

  • Measure of the differences between probability distributions

  • Minimization leads to approximation of model predictions to true class distributions

  • Promotes high probabilities for true classes and low probabilities for false classes

  • $\text{Loss} = − \sum\limits_{i=0}^{n} (y⋅log(p)+(1−y)⋅log(1−p))$

Why use a CNN to train ML models on the Fashion-MNIST dataset? Judith Romer
  • Trained to identify the edges of objects in any image
  • Can process large amounts of data and produce highly accurate predictions
  • Can learn the features of an object through multiple iterations, eliminating the need for manual feature engineering tasks like feature extraction
  • Very effective in reducing the number of parameters without losing on the quality of models, which makes it suitable for handling images because of their high dimensionality
  • Plays an important role in classification of images and image segmentation due to its great performance in different problems

Hyperparameters Judith Romer

Learning Rate

  • Affects weight updates after each training step
  • Controls loss gradient backtracking during weight updates
  • Range: learning_rate ∈ (0, ∞); lower bound: 1e-6; upper bound: 1e-2
  • Too low learning rate: slow convergence
  • Too high learning rate: divergence
  • Optimal learning rate: fast convergence with few iterations
  • Adaptive methods adjust learning rate based on accuracy or gradient
  • Non-adaptive methods use constant or stepwise learning rate schedules
  • Cyclic learning rate: varies cyclically between predefined thresholds
  • Goal: Better balance between accuracy and loss

  • Johny, A. and Madhusoodanan, K.N., 2021. Dynamic learning rate in deep CNN model for metastasis detection and classification of histopathology images. Computational and Mathematical Methods in Medicine, 2021, pp.1-13.
  • Bartz, E., Bartz-Beielstein, T., Zaefferer, M. and Mersmann, O., 2023. Hyperparameter Tuning for Machine and Deep Learning with R: A Practical Guide (p. 323). Springer Nature.
Epochs Judith Romer
  • Determines the number of iterations of the learning algorithm over the entire training dataset
  • Each example in the dataset is given an opportunity to update the model parameters
  • Includes one or more batches
  • One run over the whole dataset is organised in epochs
  • Small number of epochs: network is sensitive to changes
  • Large number of epochs: convergence and minimisation of model error, better for models quality, but not good for runtime
  • Range: epochs ∈ [1, ∞)

  • Brownlee, J., 2018. What is the Difference Between a Batch and an Epoch in a Neural Network. Machine Learning Mastery, 20.
  • Bartz, E., Bartz-Beielstein, T., Zaefferer, M. and Mersmann, O., 2023. Hyperparameter Tuning for Machine and Deep Learning with R: A Practical Guide (p. 323). Springer Nature.
Batch Size Judith Romer
  • Size of the subset of the training dataset that is used simultaneously when determining the gradient of the network weights
  • Enables parallel weight updates in training
  • Affects the runtime of the training and quality of the model
  • Smaller batch_size: increased noise in training, might hinder the learning process
  • Larger batch_size: more memory and processing power required
  • Selection of the optimal batch size requires trade-off between noise and resources
  • Range: batch_size ∈ (1, n ]

  • Brownlee, J., 2018. What is the Difference Between a Batch and an Epoch in a Neural Network. Machine Learning Mastery, 20.
  • Bartz, E., Bartz-Beielstein, T., Zaefferer, M. and Mersmann, O., 2023. Hyperparameter Tuning for Machine and Deep Learning with R: A Practical Guide (p. 323). Springer Nature.
L1 Architecture Parameter (Linear Layer Units) Judith Romer
  • Tuning the linear, fully connected layers of the CNN
  • Determines the size of the corresponding network layer
  • Significant impact on the network's performance
  • Linear layers are typically responsible for making final predictions or decisions
  • High value for l1: complex model, more model coefficients, higher runtime, but possibly higher model quality and higher risk of overfitting
  • Range: units ∈ [1, ∞)

  • Bartz, E., Bartz-Beielstein, T., Zaefferer, M. and Mersmann, O., 2023. Hyperparameter Tuning for Machine and Deep Learning with R: A Practical Guide (p. 323). Springer Nature.
Neural Architecture Search vs. Classic Hyperparameter Tuning
Hanna Steinwender

Neural Architecture Search:

  • automated process of discovering the optimal neural network architecture for a specific machine learning task
  • usage of search algorithms to explore a predefined search space of possible architectures and find the one that performs best on a given dataset and task

Classic Hyperparameter Tuning:
  • the process of systematically searching for the best hyperparameters for a machine learning model, typically after a fixed model architecture has been chosen
  • settings that control aspects of the model's training and behavior but are not learned from the data
  • main focus of this project (learning rate, epochs, batch size)

  • White, C., Safari, M., Sukthanker, R., Ru, B., Elsken, T., Zela, A., Dey, D. and Hutter, F., 2023. Neural architecture search: Insights from 1000 papers. arXiv preprint arXiv:2301.08727.
  • Tuba, E., Bačanin, N., Strumberger, I. and Tuba, M., 2021. Convolutional neural networks hyperparameters tuning. In Artificial intelligence: theory and applications (pp. 65-84). Cham: Springer International Publishing.
  • Bartz, E., Bartz-Beielstein, T., Zaefferer, M. and Mersmann, O., 2023. Hyperparameter Tuning for Machine and Deep Learning with R: A Practical Guide (p. 323). Springer Nature.

Tuning Approaches Hanna Steinwender

What different behaviors can be observed in the approaches?

Random Search

randomly and repeatedly choosing hyperparameters to evaluate

  • model-free undirected search approach
  • baseline to compare with other approaches in experiment setup
  • preferable to Grid Search for high-dimensional spaces and heterogenous parameter importance
  • Bartz, E., Bartz-Beielstein, T., Zaefferer, M. and Mersmann, O., 2023. Hyperparameter Tuning for Machine and Deep Learning with R: A Practical Guide (p. 323). Springer Nature.
  • Bergstra, J. and Bengio, Y., 2012. Random Search for Hyper-Parameter Optimization. Journal of machine learning research, 13(2).
Hyperband Algorithm Hanna Steinwender

multi-armed bandit strategy that dynamically allocates resources to a set of random configurations and uses successive halving to stop poorly performing configurations

  • model-free undirected search approach
  • employs early stopping criteria (e.g., validation loss improvement) during training
  • allocates more resources (e.g., training epochs) to promising configurations and terminates poorly performing ones early
  • efficiently identify the best-performing configurations within a given budget, avoiding wasteful computation
  • Li, L., Jamieson, K., DeSalvo, G., Rostamizadeh, A. and Talwalkar, A., 2017. Hyperband: A Novel Bandit-Based Approach to Hyperparameter Optimization. The journal of machine learning research, 18(1), pp.6765-6816.
  • Li, L., Jamieson, K., Rostamizadeh, A., Gonina, E., Hardt, M., Recht, B. and Talwalkar, A., 2018. Massively Parallel Hyperparameter Tuning.
Surrogate Model Based Optimization (SMBO) Hanna Steinwender

using the Sequential Parameter Optimization Toolkit (SPOT)

  • model-based directed search
  • information gathered during search to learn relationship between hyperparameters/performance
  • constructs a surrogate and looks for its minimum
  • computational effort can be shifted from real evaluations to evaluations of the surrogate
  • understanding hyperparameter importance and interactions between several hyperparameters

  • Bartz, E., Bartz-Beielstein, T., Zaefferer, M. and Mersmann, O., 2023. Hyperparameter Tuning for Machine and Deep Learning with R: A Practical Guide (p. 323). Springer Nature.
  • Bartz-Beielstein, T., 2023. PyTorch Hyperparameter Tuning - A Tutorial for SpotPython. arXiv preprint arXiv:2305.11930.

Providing a Summary and Key Takeaways, as well as offering Insights and Recommendations for Future Work

Key Takeaways Luisa Ibele
  • Project shows the importance of fine-tuning for achieving better performance
  • Utilization of three distinct approaches - Random Search, Hyperband, and SMBO
  • Fashion-MNIST is a valuable benchmark dataset
  • learning_rate is a very sensitive hyperparameter
  • The kind of sampling for the hyperparameter plays a role
  • Mostly no general accepted default hyperparameters for DNNs
  • Risk of Overfitting
Outlook Luisa Ibele
  • Looking at further parameters like optimizer or dropout
  • Consider other metrics like precision, recall, F1-score or the confusion matrix
  • Explore techniques like Neural Architecture Search or different Machine Learning models
  • Assess the transferability of insights gained from the hyperparameter tuning on Fashion MNIST to other image classification datasets
